# Connectors And Existing Context

Use this reference when the user mentions prior ADRs, design docs, tickets,
meeting notes, or connected knowledge sources.

## Source Priority

Prefer sources in this order when available:

1. Accepted ADRs and final architecture docs.
2. Current technical design docs.
3. Project tracker epics or implementation tickets.
4. Incident reviews and operational runbooks.
5. Meeting notes and chat threads.

## How To Use Context

- Search for prior decisions before proposing a conflicting new one.
- Cite relevant existing decisions and explain whether they still apply.
- Preserve source links when available.
- Surface conflicts between old and new documents.
- Do not assume a chat discussion is final if an accepted ADR or final design
  says something different.
